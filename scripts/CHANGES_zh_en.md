This document describes all modifications made to the original Meteor Client source code for Chinese localization support. All code and documentation were generated by AI.

本文档描述了为支持中文汉化而对 Meteor Client 原始源码所做的全部修改。所有代码和文档均由 AI 生成。

## 1. Translation Infrastructure / 翻译运行时基础设施

### Files modified:
- `src/main/java/meteordevelopment/meteorclient/utils/Utils.java`
- `src/main/java/meteordevelopment/meteorclient/MeteorClient.java`
- `src/main/java/meteordevelopment/meteorclient/commands/Command.java`
- `src/main/java/meteordevelopment/meteorclient/gui/themes/meteor/MeteorGuiTheme.java`

### Changes:

**Utils.java** — Added three static members:

| Member | Purpose |
|--------|---------|
| `TRANSLATIONS` (`HashMap<String,String>`) | In-memory translation map |
| `loadTranslations()` | Reads `zh_cn_full.json` from classpath, populates the map |
| `tr(String)` | Returns translated string, falls back to English |
| `trIfPresent(String)` | Returns translated string or `null` |

**MeteorClient.java** — `onInitializeClient()` now calls `Utils.loadTranslations()` BEFORE `ReflectInit.registerPackages()`, so that modules/settings are constructed with translated strings from the start.

**Command.java** — Constructor wraps `title` and `description` with `Utils.tr()`.

**MeteorGuiTheme.java** — The following widget factory methods wrap their `String` parameter with `Utils.trIfPresent()`:
- `window()`, `label()`, `horizontalSeparator()`, `button()`, `confirmedButton()`, `tooltip()`, `section()`

This ensures all GUI text elements display Chinese without needing per-call changes.

## 2. Translation Data / 翻译数据

### Files added:
- `scripts/translations.json` — Master translation table (3210 entries)
- `src/main/resources/assets/meteor-client/lang/zh_cn_full.json` — Runtime language file (generated)

### Files modified:
- `scripts/translate.py`
- `.github/workflows/translate.yml`

### Workflow:

1. `translate.py` scans all 949 Java source files, extracts 3213 unique English strings via regex patterns
2. Loads `translations.json` (hardcoded translations, built offline by AI)
3. For any string not found in the table, falls back to Google Translate API
4. Writes output to `zh_cn_full.json`

### Translation Statistics:

| Source | Count |
|--------|-------|
| Hardcoded table (direct) | ~2680 |
| Pattern-based rules (prefix/suffix) | ~510 |
| Google API fallback | ~22 |

### Pattern Rules (prefixes/suffixes applied during table generation):

Prefixes: `Automatically→自动`, `Disables→禁用`, `Enables→启用`, `Allows→允许`, `Prevents→防止`, `Shows→显示`, `Renders→渲染`, `How many→多少个`, `Color of→颜色`, etc.

Suffixes: `Mode→模式`, `Speed→速度`, `Color→颜色`, `Range→范围`, `Type→类型`, etc.

## 3. Font & CJK Rendering / 字体与中文渲染

### Files modified:
- `src/main/java/meteordevelopment/meteorclient/renderer/text/Font.java`
- `src/main/java/meteordevelopment/meteorclient/renderer/text/CustomTextRenderer.java`
- `src/main/java/meteordevelopment/meteorclient/systems/config/Config.java`
- `src/main/java/meteordevelopment/meteorclient/systems/hud/Hud.java`

### Changes:

**Config.java** — `customFont` default reverted to `true` (original Meteor default). Setting to `false` caused text misalignment because `VanillaTextRenderer` and `CustomTextRenderer` use different scaling calculations and font metrics.

**Hud.java** — Same `customFont` default reverted to `true`.

**Font.java** — Added `CJK Unified Ideographs` glyph range (U+4E00–U+9FFF, 20992 codepoints) to STB TrueType packing. When a CJK-capable font is loaded, these glyphs will be rendered through the custom renderer.

**CustomTextRenderer.java** — Added `containsCJK()` heuristic: if text contains characters in CJK ranges (U+3400–U+4DBF, U+4E00–U+9FFF, U+F900–U+FAFF) or any `Character.isIdeographic()` codepoint, the render call is delegated to `VanillaTextRenderer.INSTANCE`. This applies to both `getWidth()` and `render()`.

This hybrid approach:
- English/Latin text → custom font (Comfortaa, preserves original visual style)
- Chinese/CJK text → vanilla Minecraft font (correct CJK glyphs, no boxes/spaces)

## 4. Build & CI / 构建与持续集成

### Files modified:
- `.github/workflows/translate.yml`

Workflow is manually triggered (`workflow_dispatch`). Steps:
1. Checkout
2. Setup Python
3. `pip install deep-translator`
4. `python scripts/translate.py` (generates `zh_cn_full.json` from `translations.json`)
5. Setup JDK 25 + Gradle
6. `./gradlew build`
7. Upload JAR artifact
